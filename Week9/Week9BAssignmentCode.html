Introduction

This assignment require us to select one of the NYT endpoints, authenticate securely using an API key, make a live request from R, parse the JSON response, and deliver a clean, tidy data frame ready for analysis. The goal is not just to pull data but to ask a meaningful question and use the API response to begin answering it. Rather than a simple retrieval exercise, this is an end to end data pipeline task that mirrors what real data work looks like when consuming third party web services.

Only selected fileds will be used are stated in my introduction. This section creates the datafram books_df

books_df <- parsed$results$books |>
  select(title, author, publisher, rank, rank_last_week, weeks_on_list, description) |>
  as_tibble() |>
  mutate(
    rank = as.integer(rank),
    rank_last_week = as.integer(rank_last_week),
    weeks_on_list = as.integer(weeks_on_list)
  )

Conclusion

The data returned by the NYT Books API was largely clean and required minimal intervention. The response JSON is nested, with the books data living inside parsed-results-books, which required navigating two levels to parse the object before extracting it. I seleted 7 fileds total from the available fileds and those were: title, author, publisher, rank, rank_last_week, weeks_on_list, and description. The three numeric fields, rank, rank_last_week, and weeks_on_list, I changed them to integer using as.integer() since JSON parsing via jsonlite returns them as numeric by default. A rank_last_week value of zero indicates the book is newly entered on the list this week and did not hold a position the prior week, which is meaningful context rather than a missing value, so no imputation was applied.
